# ParkWiz — חבילת הצגה להאקתון

## הבעיה במשפט אחד

נהגים מבזבזים זמן ודלק בחיפוש חניה, בזמן שחניות פרטיות רבות ליד היעד עומדות פנויות בחלק משעות היום.

## הפתרון במשפט אחד

ParkWiz מחבר נהגים לחניות פרטיות פנויות ומאפשר להזמין חלון זמן מראש, ובמקביל נותן לבעלי החניה דרך להפוך זמן פנוי להכנסה.

## תסריט דמו של 3 דקות

הדמו החי: [https://swissystem7.github.io/ParkWiz/](https://swissystem7.github.io/ParkWiz/)

1. **0:00–0:30 — מציגים את הבעיה:** פותחים את הדמו ומסבירים שבמקום להסתובב ברחובות, הנהג בוחר מלאי פרטי מותר ליד היעד.
2. **0:30–1:15 — מזמינים חניה:** גוללים לאזור "הזמנת נהג", משווים בין שלוש החניות לפי מרחק ומחיר ולוחצים על "שמירת המקום בדמו".
3. **1:15–1:40 — מדגימים את העסקה:** מציגים את מזהה ההזמנה, המחיר והחזקת התשלום המדומה; לוחצים על "הדגמת שחרור אחרי החלון" ומראים שהסטטוס משתנה ל־`released`.
4. **1:40–2:25 — עוברים לצד המארח:** בטופס "פרסום חניה פרטית" מסמנים הצהרת בעלות ושולחים כדי להראות שמודעה חוקית עוברת בדיקה. אחר כך בוחרים "מקום ברחוב" ומראים שהמערכת חוסמת פרסום של חניה ציבורית.
5. **2:25–3:00 — מסכמים את הערך:** מדגישים שזה שוק דו־צדדי לנכס פרטי, שהדמו שקוף לגבי סימולציית התשלום, ושמודל ההכנסה העתידי הוא עמלת פלטפורמה.

## הטכנולוגיה

דמו Web רספונסיבי בקובץ HTML יחיד, עם JavaScript בצד הלקוח, בדיקות מדיניות מובנות ואירוח ב־GitHub Pages.

## השוק והפוטנציאל

- שוק דו־צדדי: נהגים מקבלים ודאות קרוב ליעד, ובעלי חניה מייצרים הכנסה מנכס שכבר קיים.
- מודל ההכנסה המתוכנן בריפו הוא עמלת פלטפורמה של **15%**; בדמו מוצגת עמלת השקה של 0% ל־80 המארחים הראשונים.
- אפשר להתחיל באזור צפוף אחד, למדוד הזמנות חוזרות וזמן חיפוש שנחסך, ואז להתרחב עיר־עיר בלי לפרוס חומרה ברחוב.

## מה הלאה

1. לחבר מאגר חניות, משתמשים וזמינות אמיתיים ולבצע פיילוט עם בעלי חניה באזור ממוקד.
2. להוסיף סליקה, אימות בעלות, ביטולים, דירוגים ותהליך טיפול במחלוקות.
3. למדוד זמן חיפוש שנחסך, שיעור המרה והכנסה לחניה ולהשתמש בתוצאות להתרחבות לעיר נוספת.
